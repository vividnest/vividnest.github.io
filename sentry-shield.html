Transparency

Why we need these permissions.

Sentry Shield only requests what it needs. Here's exactly what each permission is used for.

Permission Why it's needed
storageSave your block lists, settings, and timers locally on your device
tabs / activeTabDetect which site you're visiting to apply blocking rules
declarativeNetRequestBlock network requests to sites on your list at the browser level
alarmsRun scheduled blocking and reset your timers automatically
scriptingInject the blocked page when a restricted site is visited
windowsTrack the active window accurately for timer purposes
webNavigationDetect page navigations to enforce blocking even on redirects
<all_urls>Required to intercept requests on any site you choose to block
FAQ

Common questions.

Things people ask before installing.

  • Is Sentry Shield free? Yes — the core extension is completely free. Future premium features will be optional.
  • Does it collect my browsing data? No. Your block lists and settings are stored locally on your device only.
  • Will it slow down my browser? No. Sentry Shield uses the browser's native declarativeNetRequest API for blocking, which is fast and efficient on both Chrome and Firefox.
  • What happens if I forget my password? You can recover access using the security question you set during setup.
  • Can I use it on multiple computers? You can export your settings as a backup file and import them on any other device.
  • Does it block on all sites including HTTPS? Yes. Blocking works across both HTTP and HTTPS sites.
  • Will my settings survive a browser restart? Yes. All settings are saved to local storage and persist across sessions and restarts.
